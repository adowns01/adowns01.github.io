Optimizing Your SQL Queries

With a small database, the difference between optimized queries and non-optimized queries is typically so small that the user doesn't even notice. But as databases become exponentially larger, the time difference between good code and bad code increases exponentially too. Here are some basic tips to keep in mind to keep your queries quick.

Simplify your queries

# don't do this
SELECT name FROM list_name
WHERE income + 20000 > 40000

# do this instead
SELECT name FROM
WHERE income > 20000

# don't do this
SELECT name FROM list_name
WHERE income > 40000 AND income < 100000

# do this
SELECT name FROM list_name
WHERE income BETWEEN 40000 AND 100000



Use column names instead of *

Don't print out every column unless you have to

# don't do this
SELECT * FROM list_name

# do this
SELECT name, income FROM list_name


Don't use full wildcards unless absolutely necessary

# don't do this
SELECT first_name FROM list_name
WHERE first_name = '%Ame%'

# do this
SELECT first_name FROM list_name
WHERE first_name = 'Ame%'

# don't do this
SELECT last_name FROM list_name
WHERE last_name = '%wns%'

# do this
SELECT last_name FROM list_name
WHERE last_name = '%wns'



Don't use COUNT when you can use EXIST

# don't do this
SELECT name FROM list_name
WHERE COUNT(num_pets) > 0

# do this
SELECT name FROM list_name
WHERE EXISTS(num_pets)
