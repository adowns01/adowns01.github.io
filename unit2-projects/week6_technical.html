Object Oriented Design vs. Functional Design

What Is Object-Oriented Programming?

I heard about Object Oriented Programming long before I know what it was. My CS-major friends would talk about it and I often saw it on the list of requirements for jobs I applied to. I didn't know what it was, I just knew that I didn't know how to do it. I built it up in my mind, so by the time we actually covered it in my C++ class, I was shocked by how *not* complicated the topic actually is.

Object-Oriented Programming (OOP) is programming that uses objects (like classes) to design programs. Some benefits to OOP are:

  • Makes the program easier to read
  • It keeps unrelated data separate.
  • It is easier to update code.

For example here is a cat and a polygon not in classes:

cat_name = "Jason Paw-lick"
cat_age = 2

def meow(name)
 puts "#{name} says meow"
end

polygon_type = "Hexagon"
polygon_sides = 6

def print_sides (sides)
 puts "I have #{sides} sides."
end

But because they are not encapsulated in objects I can start doing some weird things that don't make sense. For example:

meow(polygon_type) #=> "Hexagon says meow"

print_sides(cat_age) #=> "I have 2 sides."

Neither of those makes sense. A polygon can't meow (sadly) and Jason Paw-lick the cat has more than 2 sides.

If these items were classes, these mishaps wouldn't be possible. For example:

class Cat
 def initialize(name, age)
  @name = name
  @age = age
 end

 def meow
  puts "#{@name} says meow"
 end
end

class Polygon
 def initialize (type, sides)
  @type = type
  @sides = sides
 end

 def print_sides
  puts "I have #{@sides} sides."
 end
end

Now that these two objects are in totally different classes so you can't program a polygon to meow or a cat to have a certain number of sides.

What is Functional Programming?

Unlike OOP, I had not heard of functional programming before reading this blog post prompt.

Functional Programming (FP) is when functions (not objects) are the building blocks of the program. Specifically, very short concise functions. FP also doesn't believe in changing values once they have been assigned.

Don't update, create.

Once a variable is created. Don't change it. If you have to change it, just create a new variable. For example:

# Don't do this
nums = [1,2]
nums << [3,4]
nums #=> [1,2,3,4]

# Do this instead
nums = [1,2]
more_nums = [nums] + [3,4]
more_nums #=> [1,2,3,4]
