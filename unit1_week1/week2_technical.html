What is the difference between Margin, Border, and Padding?

I think it's easier to think of these terms in relationship to a photograph.

The margin is located outside of the object itself and it regulates how close other objects can get to it. In terms of my metaphor, it is the space on the wall between the photo and everything else hanging on the wall. Without a margin the photos (or their frames) would be flush.

The border is the frame, it outlines the object and adds to its width and height. Just as there are different kinds of frames, there are different kinds of borders.You have a choice of solid, dashed, and more.

Padding is the mat. It is the space between the content (the photo) and the border (the frame).

What are the best practices associated with using classes vs. ids?

Objects can have multiple classes and one class can be used on multiple objects.

Objects can only have one id and a specific id can only be used once per page. Ids are ideal for headers and footers.

What are the differences between relative, absolute, and fixed positioning?

Relative positioning moves an object a specified distance from where it would normally fall. For instance. If a square had relative positioning and "right: 100px" then it would move to the right 100px from where it normally would have fallen.

Absolute positioning is relative to the whole page. So if a square had absolute positioning and "right:100px" it would appear 100px from the right handside of the screen.

Fixed positioning is relative to the browser. So if a square had fixed positioning and is "top; 100px" then regardless of scrolling, the item would always appear 100px from the top of your browser.

What does it mean to display inline vs inline blocks?

Inline elements go with the flow of the website; they don't start on a new line. Some elements are automatically inline, like bolding or italicising. When an item is bolded it doesn't start on a new line from the other words in the sentence. When an item is inline, you can't specify or height or width for it.

Inline-block elements can have height and width values and still goes with the flow of the website.
